# jingbanyou-tourist 日志规范

**制定日期**: 2026-05-12
**适用范围**: jingbanyou-tourist 模块所有 Java 类

---

## 1. 日志级别定义

| 级别 | 说明 | 使用场景 |
|------|------|----------|
| **ERROR** | 系统错误，需要人工介入 | AI 服务调用失败、TTS 合成失败、数据库写入失败、外部 API 调用异常 |
| **WARN** | 异常但可降级处理 | 缓存过期、限流触发、熔断开启、非关键配置缺失 |
| **INFO** | 关键业务流程节点 | 请求开始/结束、Graph 执行耗时、缓存命中、会话同步完成 |
| **DEBUG** | 调试信息，线上默认关闭 | Redis 读取 miss、参数解析细节、中间状态输出 |

## 2. 各服务日志规范

### 2.1 EmotionDetectServiceImpl

| 场景 | 日志级别 | 原因 |
|------|----------|------|
| 情感检测整体失败 | ERROR | AI 服务调用异常，可能影响游客画像分析 |
| Redis 读取情感结果失败 | DEBUG | 预期的缓存 miss，无需告警 |
| AI 分析失败降级为规则 | WARN | 降级可接受，但需关注降级频率 |
| 检测成功写入 Redis | INFO | 关键业务节点 |
| 删除情感结果失败 | WARN | 清理失败非关键，但可能积累脏数据 |

| LLM 流异常 | ERROR | 核心功能失败 |

| 删除对话元数据失败 | WARN | 清理失败可能积累脏数据 |

## 3. 通用规则

1. **ERROR 必须触发告警**：所有 ERROR 级别日志应配置监控告警
2. **INFO 记录关键参数**：含 sessionId、visitorId 等上下文信息
3. **异常必须打印堆栈**：ERROR 级别异常使用 `log.error(msg, e)` 传异常对象
4. **WARN 需定期检查**：降级类 WARN 应统计频率，异常增高时应升级为告警
5. **DEBUG 预留上下文**：DEBUG 日志应包含足够信息用于问题排查
