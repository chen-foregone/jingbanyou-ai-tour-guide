# 新技术学习笔记

## 1. @Async — Spring 异步注解

### 作用
把方法调用扔到 Spring 线程池执行，调用线程立即返回，不阻塞。

### 本质
底层就是 `ThreadPoolExecutor`，方法在工作线程中同步执行。

```java
@Async
public void detectEmotionAsync(String msg) {
    String result = chatClient.call(msg); // 工作线程阻塞等 HTTP 响应
}
```

```
主线程：调用 → 立即返回（不等）
工作线程：执行 → 阻塞等 HTTP → 完成
```

### 局限
内部代码仍然同步执行，HTTP 请求仍然阻塞工作线程。高并发时线程池耗尽。

## 2. CompletableFuture.runAsync — Java 标准库

### 作用
把 `Runnable` 扔到线程池执行，不阻塞调用线程。

```java
CompletableFuture.runAsync(() -> {
    // 新线程执行
});
```

### 对比 @Async

| | `@Async` | `CompletableFuture.runAsync` |
|--|---------|----------------------------|
| 来源 | Spring 注解 | Java 标准库 `java.util.concurrent` |
| 线程池 | Spring 线程池（可配置） | 可指定，或默认 `ForkJoinPool.commonPool()` |
| 返回值 | void | `CompletableFuture<Void>` |
| 适合场景 | Spring bean 完整方法 | 任意代码片段（轻量调用） |

### 指定线程池

```java
ExecutorService pool = Executors.newFixedThreadPool(5);

CompletableFuture.runAsync(() -> {
    // 在 pool 线程执行
}, pool);
```

## 3. Spring AI Graph 异步节点

### 配置方式

```java
// 每个节点用 AsyncNodeAction.node_async() 包装
stateGraph.addNode("myNode", AsyncNodeAction.node_async(myNodeAction));
```

### 实际行为

- 节点在**独立线程**执行，不阻塞 graph 主线程
- **节点之间仍然是串行的**（先 ProfileLoader → 等完成 → 再 TextDistinguish → 等完成 → ...）
- 不是"多分支并行"，条件路由只选一条边

### 流程示意

```
graph.invoke()
  → Thread-A: ProfileLoader.apply() 完成
  → graph 分发
  → Thread-B: TextDistinguish.apply() 阻塞等 HTTP 完成
  → graph 分发
  → Thread-C: MapRouteApiInvoker.apply() ...
```

## 4. 同步阻塞 vs 非阻塞 I/O

### 同步阻塞（默认 HTTP 客户端）

`RestTemplate`、`OkHttp`（默认模式）、`URLConnection` 都是同步阻塞：

```java
// 主线程/工作线程被挂起，等 TCP socket 响应
String result = restTemplate.getForObject(url, String.class);
```

### 真正非阻塞 I/O

`WebClient`（Spring WebFlux）+ 响应式编程：

```java
webClient.get()
    .uri(url)
    .retrieve()
    .bodyToMono(String.class)  // 非阻塞，返回 Mono
    .subscribe(result -> {
        // 响应到达后在此回调，不占用任何线程等待
    });
```

底层基于 Netty（事件驱动），用很少的线程处理大量并发。

## 5. Fire-and-Forget 模式

### 适用场景
任务不需要等待结果，不影响主流程，只是后台记录。

### 代码示例

```java
// 主流程继续，不需要等情感分析完成
CompletableFuture.runAsync(() -> {
    emotionDetectService.detectEmotionAsync(sessionId, message, null);
}, dedicatedPool);

return graph.invoke(initialState);  // 立即执行
```

### 结果存储
情感分析完成后写入 Redis → `endChat` 时读取 → 写入 MySQL。

## 6. 线程池配置建议

```java
@Configuration
@EnableAsync
public class AsyncConfig {
    @Bean("emotionPool")
    public Executor emotionPool() {
        return new ThreadPoolExecutor(
            2,                      // 核心线程数（景区访问量小，2个够用）
            5,                      // 最大线程数
            60L, TimeUnit.SECONDS,   // 空闲回收
            new LinkedBlockingQueue<>(100),
            new ThreadFactory("emotion-")
        );
    }
}
```

使用时指定线程池：

```java
CompletableFuture.runAsync(() -> {
    // ...
}, emotionPool);  // 注入上面的 bean
```
